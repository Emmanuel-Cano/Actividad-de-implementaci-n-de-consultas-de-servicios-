### **Ciclo de Vida del Desarrollo de Software**

#### **Proceso estrucrurado**
Para planear, crear, probar y mantener un sistema o aplicación. Esta compuesto por varias etapas que permiten asegurar la calidad y el cumplimiento de los requisitos del cliente.
1. Recolección de Requisitos.
2. Análisis de Requisitos
3. Diseño
4. Pruebas
5. Implementación
6. Mantenimiento
---

1. Recolección de requisitos (Análisis)
¿Qué se hace? Se identifica lo que el cliente necesita.
Objetivo: Entender el problema o necesidad a resolver.
Ejemplo: El cliente quiere una app para llevar control de inventario.

2. Análisis de requisitos
¿Que se hace? Se analiza la información obtenida para definir exactamente qué debe hacer el sistema.
Objetivo: Establecer requisitos funcionales y no funcionales.
Ejemplo: El sistema debe regitrar productos, generar reportes y enviar alertas.

3. Diseño del Sistema
•¿Qué se hace? Se define cómo funcionará el software internamente.
•Objetivo: Crear diagramas, estructuras de datos y arquitectura del sistema.
•Ejemplo: Diagrama de flujo, modelo entidad-relación (base de datos), arquitectura en capas.

4. Desarrollo (Codificación)
•¿Qué se hace? Se escribe el código fuente del programa usando un lenguaje de programación.
•Objetivo: Convertir el diseño en un sistema funcional.
•Ejemplo: Programar los formularios, bases de datos y funcionalidades.

5. Pruebas (Testing)
•¿Qué se hace? Se revisa que el software funcione correctamente y sin errores.
•Objetivo: Detectar y corregir fallos antes de entregarlo al cliente.
•Ejemplo: Pruebas de unidad, pruebas de integración y pruebas de aceptación

6. Implementación (Despliegue)
•¿Qué se hace? Se entrega el software al cliente o se publica en un entorno real.
•Objetivo: Poner el sistema en funcionamiento.
•Ejemplo: Instalar la aplicación en los equipos del cliente o subirla a la nube.

7. Mantenimiento
•¿Qué se hace? Se corrigen errores o se agregan mejoras después de la entrega.
•Objetivo: Asegurar que el sistema siga funcionando correctamente con el tiempo.
•Ejemplo: Corregir un bug, actualizar funciones o adaptar a nuevas necesidades.

### **Tipos de sistemas de información** ###
##### **TPS, MIS, DSS, EIS, CRM, ERP**
###### **TPS- Transaction Processing System (Sistema de Procesamiento de Transacciones)**
¿Qué es?
Sistema encargado de registrar y procesar transacciones diarias de una organización, como ventas, compras, pagos, etc.
Características:
•Alto volumen de datos.
•Precisión y velocidad.
•Procesamiento automático y rutinario
•Interacción directa con usuarios.

Casos reales:
•Visa: procesa millones de transacciones de tarjetas de crédito.
•Walmart: usa TPS para registrar ventas y actualizar inventario en tiempo real.

Ejemplos de uso:
Cajeros automaticos (ATM)
Punto de venta (POS) en supermercado

###### **MIS - Management Information System (Sistema de Información Gerencial)**
¿Qué es?
Sistema que convierte los datos del TPS en información útil para la toma de decisiones gerenciales.
Características:
•Genera reportes periódicos (diarios, semanales, mensuales).
•Soporte a niveles medios de administración.
•Se basa en datos históricos y actuales.
Ejemplos de uso:
•Reportes de ventas mensuales.
•Análisis de rendimiento por sucursal.
Casos reales:
•Coca-Cola: usa MIS para monitorear el desempeño por región y ajustar campañas de marketing.
•FedEx: genera informes de eficiencia operativa.

###### **DSS - Decision Support System (Sistema de Soporte a Decisiones)**
¿Qué es?
Sistema que ayuda a tomar decisiones complejas y no rutinarias mediante análisis de datos, simulaciones y modelos.
Características:
•Analiza grandes volúmenes de datos.
•Ofrece escenarios hipotéticos ("¿qué pasaría si...").
Interactivo y flexible.
Ejemplos de uso:
•Planificación de rutas logísticas.
•Análisis de inversiones.
Casos reales:
•Amazon: usa DSS para optimizar inventarios y logística según demanda proyectada.
•Delta Airlines: utiliza DSS para programar vuelos y administrar precios dinámicos.

###### **EIS (Executive Information System) o Sistema de Información Ejecutiva** ######

Es un tipo de sistema de información diseñado para ayudar a los altos directivos o ejecutivos de una empresa a tomar decisiones estratégicas de forma rápida y eficaz.
Características
1. Orientado a ejecutivos
2. Información resumida
3. Acceso rápido a datos
4. Visualización de KPIs
5. Análisis de tendencias
6. Alertas y notificaciones
Descripción
Diseñado para gerentes y directivos, no para personal operativo.
Presenta datos clave en forma gráfica y sencilla.
Permite consultar información de distintas áreas en segundos.
Muestra indicadores como ventas, utilidades, crecimiento, etc.
Ayuda a identificar patrones y oportunidades de mejora.
Informa sobre desviaciones o problemas en tiempo real.
Empleos y Casos Reales
General Electric (GE)
•GE implementó un EIS para que sus directivos pudieran visualizar el desempeño de todas sus
unidades de negocio en tiempo real.
•Esto les permitió tomar decisiones rápidas sobre inversiones, reducción de costos y planificación global.

###### **CRM (Customer Relationship Management)**
Es un sistema que permite a las empresas gestionar las relaciones con sus clientes. Ayuda a organizar, automatizar y sincronizar las áreas de ventas, marketing, atención al cliente y
soporte postventa.

Característica
1. Gestión de clientes
2. Seguimiento de ventas
3. Automatización de marketing
4. Atención al cliente
5. Historial de interacciones

Descripción
Almacena toda la información de cada cliente.
Permite ver en qué etapa del proceso está cada venta.
Envío de correos automáticos, campañas segmentadas.
Registra solicitudes, quejas o reclamos.
Guarda llamadas, correos, reuniones, etc.

Caso real (CRM):
Spotify utiliza CRM para:
-Analizar los gustos musicales de los usuarios.
•Crear recomendaciones personalizadas.
•Enviar correos con playlists sugeridas según sus hábitos.

###### **ERP (Enterprise Resource Planning)** 
Es un sistema que integra todos los procesos de una empresa en un solo sistema: finanzas, recursos humanos, compras, inventario, producción, ventas, etc.

Característica
1. Integración total
2. Base de datos única
3. Automatización de procesos
4. Reportes financieros y operativos
5. Modularidad

Descripción
Conecta todas las áreas de la empresa en una misma plataforma.
Toda la información se actualiza en tiempo real y es accesible por todos.
Reduce errores humanos y tareas manuales.
Permite tomar decisiones rápidas y con
datos confiables.
Se pueden implementar solo los módulos necesarios (ej. contabilidad, inventario, etc.).
